# Harden PR5: serde convenience layer

**Date**: 2026-03-04
**Branch**: `pr5-serde`

## Changes Made

### Bug Fixes
1. **Missing `TimeoutError` in Python/TS clients** ‚Äî Added `TimeoutError` class to `errors.py` and `errors.ts`, plus `-5` handling in `check_error`.
2. **FFI register returned wrong error code** ‚Äî `rebar_register` now returns `REBAR_ERR_ALREADY_REGISTERED (-6)` for duplicate names instead of `REBAR_ERR_NOT_FOUND`. Added `AlreadyRegisteredError` to all client libraries.
3. **`send_named` fabricated fake PID** ‚Äî Replaced `SendError::ProcessDead(ProcessId::new(0,0))` with new `SendError::NameNotFound(String)` variant.
4. **Integration tests used `.await` on now-sync `send()`** ‚Äî Fixed all `.await` calls on `ProcessContext::send()` and `Runtime::send()` across integration tests.
5. **FFI recv race condition** ‚Äî `rebar_recv` previously removed `MailboxRx` from the map during recv, causing concurrent callers on the same PID to get `NOT_FOUND`. Now uses `Arc<Mutex<MailboxRx>>` per PID for safe concurrent access.

### Refactors
6. **Deduplicated deserialization logic** ‚Äî `recv_typed` and `recv_typed_timeout` now delegate to `Message::deserialize()` instead of duplicating the Binary match + deserialize pattern.

### Tests Added
7. `send_typed_recv_typed_roundtrip` ‚Äî serde round-trip through typed send/recv
8. `recv_typed_timeout_succeeds` ‚Äî typed recv with timeout
9. `recv_typed_non_binary_returns_none` ‚Äî non-binary payload returns None
10. `spawn_with_bounded_mailbox` ‚Äî bounded mailbox rejects at capacity
11. `register_whereis_unregister` ‚Äî name registry lifecycle
12. `register_duplicate_name_fails` ‚Äî duplicate registration error
13. `send_named_delivers_message` ‚Äî send via registered name
14. `send_named_unknown_name_returns_error` ‚Äî NameNotFound error
15. `is_alive_and_kill` ‚Äî process lifecycle APIs
16. `list_processes` ‚Äî process enumeration
